## Recommended Approach

### Option A: Interactive (Recommended)

```bash
rails app:update
```

For each file, you'll be prompted:
- **Y** - Accept the change
- **n** - Keep your version
- **d** - View the diff
- **q** - Quit

**Follow this decision tree:**

```
Is it a new file?
  → Accept (Y)

Is it config/application.rb load_defaults?
  → Accept (Y)

Is it a file you've customized?
  → View diff (d), then decide

Is it a bin/ script?
  → Usually accept (Y)

Not sure?
  → View diff (d), compare carefully
```

### Option B: Manual (For Complex Apps)

1. Generate a new Rails app with the target version
2. Compare files manually
3. Apply changes selectively

```bash
# Create comparison app
rails new temp_app_compare --skip-bundle
cd temp_app_compare

# Compare specific files
diff -u ../your_app/config/application.rb config/application.rb
```

### Option C: Accept All (Simple Apps Only)

```bash
rails app:update --force
```

⚠️ **Warning:** This overwrites all files. Only use if:
- You have version control
- Your config files are standard
- You can review changes afterward

## New Framework Defaults

After upgrading, Rails creates:
`config/initializers/new_framework_defaults_{TO_SLUG}.rb`

This file lets you opt-in to new behaviors gradually:

```ruby
# Don't enable all at once!
# Enable one at a time and test:

# Rails.application.config.some_new_default = true
# Rails.application.config.another_default = "value"
```

### Enable Gradually:

1. Uncomment one option
2. Run test suite
3. Test manually
4. Deploy to staging
5. Monitor for issues
6. Repeat for next option

### Once All Verified:

1. Update `config.load_defaults` to {TO_SHORT}
2. Delete the new_framework_defaults file

## Troubleshooting

### "I accidentally overwrote a file"

```bash

```

### "The app won't start after update"

Check the Rails logs:
```bash
tail -f log/development.log
```

Common issues:
- Missing gem versions
- Invalid configuration syntax
- Removed deprecated settings

### "Tests are failing after update"

1. Read the error messages carefully
2. Check for deprecated methods
3. Look for changed default behaviors
4. Consult the Rails upgrade guide
